# 상용화 전 수동 QA 체크리스트

배포 전·후에 아래 항목을 **스테이징(또는 프로덕션 소규모)** 에서 실행하고, 결과를 기록합니다.  
자동 CI(`Tests` 워크플로: PHP 문법, tsc, ESLint, 빌드)는 통과했어도, **실제 브라우저·결제·DB·권한**은 여기서만 검증할 수 있습니다.

## 기록

| 환경 | 일자 | 담당 | 비고 |
|------|------|------|------|
| 스테이징 | | | |
| 프로덕션 | | | |

---

## 1. 인증·계정

- [ ] 카카오 로그인 성공
- [ ] 콜백 후 세션/토큰 정상 (로그인 상태 유지)
- [ ] 토큰 갱신(장시간 사용 또는 401 후 재시도) 동작
- [ ] 로그아웃 후 보호된 페이지 접근 불가

## 2. 구독·페이월·기사 접근

- [ ] 비구독(또는 미로그인): 제한 기사에서 페이월/부분 노출·CTA 동작
- [ ] 구독 중: 전체 기사·오디오 등 접근
- [ ] 만료 직후(가능 시): 접근 제한 재적용

## 3. 콘텐츠·탐색

- [ ] 홈 → 기사 상세 → 북마크 → 목록 복귀
- [ ] 검색 결과 동작
- [ ] 탭/라우트 전환 후 상태 복원(예: `fromTab`)

## 4. 구독·결제(해당 시)

- [ ] 주문·검증 플로우
- [ ] 스텝페이 웹훅(스테이징 테스트 결제 1건)

## 5. 관리자

- [ ] 관리자만 관리자 페이지/API 접근
- [ ] 기사 작성·GPT 분석·저장 — 매체 설명·TTS 메타
- [ ] (베타) 기사 챗봇: **관리자만** UI 노출 및 API 동작
- [ ] 일반 계정: **챗봇 UI 없음**, `/api/article-chat.php` 호출 시 403

## 6. 운영·헬스

- [ ] `GET /api/health` (또는 배포된 사이트 health URL) 200
- [ ] 메인 페이지 200
- [ ] 장애 시 `storage/logs`, Nginx/PHP-FPM 로그 확인 경로 숙지

## 7. 장애 시나리오(선택·권장)

- [ ] 결제 실패/취소 화면
- [ ] 토큰 만료 후 재로그인 유도
- [ ] 긴 GPT 분석·SSE 스트리밍 시 타임아웃 없이 완료

---

**통과 기준**: 위 필수 항목(1~6) 중 비즈니스에 필요한 항목이 모두 OK일 것.  
**CI 참고**: `main`/`develop` push 및 `main` PR 시 `.github/workflows/test.yml`이 자동 실행됩니다. 배포 전에 해당 워크플로가 초록인지 확인하세요.
