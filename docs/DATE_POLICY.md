# 기사 날짜 정책

## 단일 기준 (표시용)

- **사용자에게 보이는 기사 날짜**: `created_at` (우리 사이트에 게시된 날짜)
- **목록·상세 모두** 동일한 기준으로 표시하여 첫 페이지와 상세 페이지 날짜가 일치하도록 함

## 필드 역할

| 필드 | 역할 |
|------|------|
| `created_at` | 표시용 기본 날짜. 우리가 기사를 게시한 날짜. |
| `updated_at` | 내부 수정 이력용. 게시일 표시에는 사용하지 않음. |
| `published_at` | 원문 기사 발행일 메타데이터. 필요 시 별도 표기용으로만 사용. |

## API 응답

- 목록·상세 API는 표시용 날짜를 **`display_date`** (= created_at)로 내려준다. 상세는 `published_at`에도 동일 값 전달.
- 프론트는 **`display_date` 우선**, 없으면 `published_at`을 사용해 날짜를 렌더링한다.

## 검증 체크리스트 (회귀 확인용)

다음 케이스에서 **홈 카드 / 전체기사 리스트 / 상세 헤더** 날짜가 동일한지 확인한다.

1. **원문 발행일이 있는 기사** (예: 스크래퍼로 수집한 기사) – 표시는 created_at 기준으로 통일되어야 함.
2. **원문 발행일이 없는 기사** – created_at만 있으므로 모든 화면에서 동일.
3. **수정 이력이 있는 기사** – 상세는 updated_at가 아닌 display_date(created_at)만 표시.
4. **대표 기사** (예: 미국발 무역 카오스) – 목록과 상세 날짜가 일치하는지 확인.
